# Skill: Code Review

## 1. 목적

Code Review 작업을 반복 가능한 절차로 수행하기 위한 Skill이다. 이 Skill은 작업 결과의 품질 편차를 줄이고 후속 담당자가 맥락을 이해할 수 있게 한다.

## 2. 사용 조건

요청된 작업이 Code Review 범위에 해당할 때 사용한다. 입력이 부족하면 작업을 시작하기 전에 assumptions 또는 확인 필요 사항으로 분리한다.

## 3. 입력

- 작업 설명
- 관련 요구사항 또는 코드 위치
- 제약사항
- 기대 산출물

## 4. 작업 절차

1. 입력과 현재 문서를 확인한다.
2. 영향 범위와 위험을 식별한다.
3. 필요한 작업을 작은 단위로 수행한다.
4. 결과와 검증 방법을 기록한다.

## 5. 검토 기준

결과가 요청 범위에 맞는지 확인한다. 누락된 정보, 모순, 테스트 공백, 보안/운영 위험이 있으면 별도로 기록한다.

## 6. 출력 형식

- 요약
- 상세 결과
- 변경 또는 제안 사항
- 검증 결과
- 남은 위험

## 7. 품질 기준

출력은 재현 가능하고 후속 작업자가 이해할 수 있어야 한다. 단정할 수 없는 내용은 추정으로 표시하고 확인 방법을 함께 남긴다.

## 8. 금지사항

승인 없이 범위를 확장하거나 관련 없는 리팩터링을 포함하지 않는다. 민감정보나 검증되지 않은 주장을 문서에 확정 사실처럼 기록하지 않는다.
