# Base Harness

이 폴더는 모든 템플릿에 복사할 수 있는 하네스 엔지니어링 기본 구조다. 하네스는 개발 작업을 실행 명령, 검증 기준, 증거 로그, 산출물 위치로 묶어 반복 가능하게 만드는 운영 장치다.

Agent는 구현 전에 이 폴더를 확인하고 현재 프로젝트에서 실제로 실행 가능한 명령과 검증 기준을 정리해야 한다. 아직 명령이 없으면 추정하지 말고 후보 명령과 확인 질문을 분리해 기록한다.

## 구성

- harness.yaml: 하네스 메타데이터와 품질 게이트
- commands.md: 설치, 실행, 테스트, 빌드, 배포 명령 목록
- verification-matrix.md: 작업 유형별 검증 방법
- evidence-log.md: 실행 결과와 증거 기록
- runbook.md: 하네스 운영 절차
- fixtures/: 테스트 데이터와 fixture 위치
- scripts/: 프로젝트별 보조 스크립트 위치
- reports/: 하네스 실행 산출물 위치
