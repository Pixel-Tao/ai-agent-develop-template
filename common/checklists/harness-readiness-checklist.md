# Harness Readiness Checklist

이 체크리스트는 프로젝트 작업 전에 실행/검증/증거 기록 체계가 준비되었는지 확인한다.

- [ ] harness/README.md가 있다.
- [ ] harness/harness.yaml에 목적과 품질 게이트가 있다.
- [ ] harness/commands.md에 실행 후보 또는 확정 명령이 있다.
- [ ] harness/verification-matrix.md에 작업 유형별 검증 기준이 있다.
- [ ] harness/evidence-log.md에 실행 결과를 기록할 수 있다.
- [ ] reports/, fixtures/, scripts/ 위치가 있다.
- [ ] 검증 불가 항목을 assumptions, blockers, risks로 분리하는 규칙이 있다.
