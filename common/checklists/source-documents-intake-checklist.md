# Source Documents Intake Checklist

이 체크리스트는 사용자가 미리 넣은 초기 개발 문서를 Agent가 안전하게 읽고 인터뷰에 활용했는지 확인한다. 원본 문서를 그대로 요구사항으로 확정하지 않고, 근거와 질문으로 분리하는 것이 핵심이다.

## 문서 수집

- [ ] inputs/README.md가 존재한다.
- [ ] inputs/source-documents-index.md가 존재한다.
- [ ] initial-development-docs/에 사용자가 제공한 원본 문서를 넣을 수 있다.
- [ ] references/에 참고 자료와 링크를 넣을 수 있다.

## 문서 해석

- [ ] 원본 문서의 출처와 작성자를 기록했다.
- [ ] 문서의 작성일 또는 최신성을 확인했다.
- [ ] 확인된 사실, 추정, 미정, 충돌 내용을 분리했다.
- [ ] 인터뷰에서 확인해야 할 질문을 도출했다.

## 인터뷰 연결

- [ ] Project Discovery Interview에서 초기 문서를 먼저 검토했다.
- [ ] 문서 기반 질문과 사용자 답변을 연결했다.
- [ ] 공식 요구사항으로 반영한 항목과 보류한 항목을 구분했다.
- [ ] project-definition-brief에 근거 문서 ID를 남겼다.
