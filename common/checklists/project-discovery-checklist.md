# Project Discovery Checklist

이 체크리스트는 프로젝트를 구현하기 전에 사용자의 막연한 아이디어가 실행 가능한 프로젝트 정의로 정리되었는지 확인한다. 모든 항목을 완벽히 채우지 못해도 되지만, 비어 있는 항목은 assumptions 또는 open questions로 남겨야 한다.

## 기본 구체화

- [ ] 사용자의 최초 아이디어 원문을 기록했다.
- [ ] 프로젝트 목표와 이번 버전 목표를 분리했다.
- [ ] 대상 사용자와 사용 상황을 구체화했다.
- [ ] 성공 기준과 검증 방법을 기록했다.

## 범위 구체화

- [ ] 반드시 포함할 기능을 정했다.
- [ ] 다음 버전으로 미룰 기능을 정했다.
- [ ] 명시적 제외 범위를 기록했다.
- [ ] 핵심 사용자 흐름을 최소 1개 이상 작성했다.

## 콘텐츠와 기술 구체화

- [ ] 핵심 콘텐츠 또는 데이터 항목을 정의했다.
- [ ] 콘텐츠/데이터 출처를 기록했다.
- [ ] 기술 스택 또는 기술 선택 기준을 기록했다.
- [ ] 외부 연동, 인증, 결제, 파일, AI 기능 필요 여부를 확인했다.

## 안전장치

- [ ] 보안/개인정보/권한 관련 위험을 확인했다.
- [ ] 운영, 로그, 장애 대응 관련 제약을 확인했다.
- [ ] 결정, 가정, 미정을 분리했다.
- [ ] project-definition-brief가 작성되었다.
