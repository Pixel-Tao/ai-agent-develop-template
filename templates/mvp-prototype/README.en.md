# MVP Prototype

Korean is the default documentation language. This file is the English companion for [README.md](README.md).

## Purpose

A lightweight structure for validating ideas and building fast prototypes with short feedback cycles.

## Use When

- You need quick validation
- The experiment cycle is short
- MVP scope is intentionally small

## Do Not Use When

- The project is regulated
- Long-term large-scale operations are required

## Initialization

1. Fill project metadata in `manifest.yaml`.
2. Define MVP scope and experiment plan.
3. Record assumptions, quick decisions, and user feedback.
4. Track current state in `docs/09_agent_state/current-status.md`.

## Required Entrypoints

- `README.md`
- `AGENTS.md`
- `manifest.yaml`
- `template.yaml`
- `validation-checklist.md`
- `docs/09_agent_state/current-status.md`
