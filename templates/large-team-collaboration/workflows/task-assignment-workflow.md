# Task Assignment Workflow

작업을 역할과 담당자에게 배정하는 절차다. 의존성, 우선순위, 승인 필요 여부를 확인한 뒤 work allocation에 등록한다.

## 사용 방법

이 문서는 프로젝트 초기화 시 초안을 작성하고 작업 진행에 따라 갱신한다. AI Agent는 작업 전에 이 문서의 현재 상태를 확인하고, 불확실한 내용은 단정하지 않고 assumptions 또는 unknowns에 기록한다.

## 절차

1. 요청의 목적과 범위를 확인한다.
2. RACI 기준으로 Responsible 역할을 정한다.
3. 의존성을 dependency board에 등록한다.
4. 담당자, 우선순위, 마감일을 work allocation에 기록한다.
5. 승인 필요 작업은 approval policy와 연결한다.
