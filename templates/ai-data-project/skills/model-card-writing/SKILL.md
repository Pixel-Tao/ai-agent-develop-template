# Skill: Model Card Writing

## 1. 목적

Model Card Writing 작업을 이 템플릿의 문서 구조와 품질 기준에 맞게 수행한다. 이 Skill은 입력 문서, 작업 절차, 산출물 형식을 통일해 Agent와 사람이 같은 기준으로 협업하게 한다.

## 2. 사용 조건

요청된 작업이 Model Card Writing 범위에 해당할 때 사용한다. 입력이 부족하면 assumptions 또는 해당 템플릿의 상태 문서에 확인 필요 사항을 기록한다.

## 3. 입력

- 작업 설명
- 관련 문서
- 제약사항
- 기대 산출물

## 4. 작업 절차

1. manifest.yaml과 AGENTS.md를 확인한다.
2. 관련 문서와 현재 상태를 읽는다.
3. 영향 범위와 위험을 기록한다.
4. 작업을 수행하고 검증한다.
5. run log와 필요한 산출물을 갱신한다.

## 5. 검토 기준

요청 범위를 벗어난 변경이 없는지 확인한다. 템플릿별 품질 게이트와 승인 조건을 충족했는지 확인한다.

## 6. 출력 형식

- 요약
- 수행 내용
- 변경 또는 생성한 문서
- 검증 결과
- 남은 위험

## 7. 품질 기준

결과는 다음 담당자가 이어받을 수 있을 정도로 명확해야 한다. 검증하지 못한 내용은 완료로 단정하지 않는다.

## 8. 금지사항

승인 없이 위험 변경을 완료 처리하지 않는다. 민감정보나 확인되지 않은 주장을 확정 사실처럼 남기지 않는다.
